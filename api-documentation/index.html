Authentication Bearer Token

All requests require a bearer token in the Authorization header. Tokens are issued per service account through the Access module.

Token lifetime defaults to 24 hours and can be refreshed via POST /v2/access/tokens/refresh.

Authorization RBAC

Role-based access control is enforced per module. A token's granted scopes appear in the token payload and are validated against every endpoint.

Error Codes

  • 400 Bad RequestMalformed payload
  • 401 UnauthorizedInvalid token
  • 403 ForbiddenScope not granted
  • 404 Not FoundResource missing
  • 429 Too Many RequestsRate limit hit
  • 500 Server ErrorContact support

Pagination Cursor

List endpoints return a cursor field. Pass it as a query parameter on the next request to retrieve the following page. Default page size is 50; max is 200.

Filtering Predicates

Filter lists with filter[field]=value syntax. Operators: eq, ne, gt, lt, gte, lte, in, like.

Rate Limiting

60 requests per minute per token. Response headers expose X-RateLimit-Remaining and X-RateLimit-Reset.

Versioning v2.0

Current stable release is v2.0. Breaking changes are introduced only in new major versions. Deprecations are announced 12 months in advance.
